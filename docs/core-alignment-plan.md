# 核心对齐规划

这份规划只描述行为映射，不在 Java 代码中保留参考项目的名称或标识。

## 第一阶段范围

先做纯 Java 聊天大脑，不接 Forge、不接战斗、不接视角摘要、不接复杂工具。

必须完成：

- 会话级运行时：每个会话一个 Runtime，维护状态、历史、待处理消息和等待状态。
- 消息缓存：玩家消息先进入缓存，静默窗口结束后批量进入一轮思考。
- 说话节奏：支持频率阈值、强制继续、等待、等待超时后重新判断。
- 打断机制：新消息到达时能取消当前可取消的模型请求，并重新组织上下文。
- 主规划器：根据历史和最新消息决定 reply、wait、no_action。
- 回复器：只生成可见台词，不输出分析、记忆、标签或结构化内容。
- 分句器：把回复拆成自然气泡片段，同时保护过短句和括号内容。
- 配置结构：分目录管理人设、模型、会话、分句、调试。

## 行为映射

| 参考核心行为 | Java 原型模块 | 说明 |
| --- | --- | --- |
| 会话管理、最近消息、会话上下文 | `ChatSession` | 保存历史、缓存、状态、强制继续标记 |
| 会话运行时和内部循环 | `ConversationRuntime` | 负责调度、打断、等待、输出 |
| 思考阶段执行 | `ReasoningEngine` | 消费消息批次，调用节奏判断、规划和回复 |
| 上下文选择 | `ContextWindow` | 只取最近有效消息，不把无关原始历史塞给模型 |
| 规划器 | `PlannerAgent` | 输出严格 JSON 决策 |
| 回复生成器 | `ReplyComposer` | 构造干净 prompt，并只返回可见台词 |
| 输出后处理 | `ReplySanitizer`、`SentenceSplitter` | 清洗格式泄漏，分割自然短句 |
| 配置热结构 | `BrainConfig` | 分目录 properties，后续可替换为模组配置页 |

## 关键约束

- 普通玩家消息进入同一个 Runtime，不允许独立模块绕过 Runtime 发言。
- 旧轮次若被新消息打断，不能继续把旧结果输出。
- 模型输出只允许是女仆最终会说出口的话。
- 所有 prompt 都在 `prompts/zh-CN`，代码只负责填充变量。
